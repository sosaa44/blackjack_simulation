1. Einleitung

Blackjack ist eines der beliebtesten Casinospiele der Welt. Im Gegensatz zu vielen anderen Glücksspielen hängt das Ergebnis nicht nur vom Zufall ab, sondern auch von den Entscheidungen des Spielers. Das wirft eine interessante Frage auf: Macht die gewählte Strategie langfristig wirklich einen Unterschied?

Fragestellung: Welche Blackjack-Strategie überlebt am längsten — und wie gross ist die Wahrscheinlichkeit, das Casino zu schlagen?

Wir haben dieses Thema gewählt, weil es einen realistischen stochastischen Prozess mit mehreren Zufallsschritten und verschiedenen Wahrscheinlichkeitsverteilungen abbildet. Die Ergebnisse sind direkt interpretierbar: In unserem Modell zeigt sich kein stabiler langfristiger Vorteil gegenüber dem Casino, aber manche Strategien schneiden deutlich besser ab als andere.

2. Modellbeschreibung

2.1 Blackjack-Regeln (vereinfacht)

Wir simulieren eine vereinfachte Version von Blackjack mit folgenden Regeln:

  • Ein Standard-Kartendeck mit 52 Karten wird verwendet (4 Farben × 13 Werte)
  • Kartenwerte: 2–9 = Zahlenwert, 10/Bube/Dame/König = 10, Ass = 11 (oder 1 bei Bust)
  • Der Dealer zieht immer bis er mindestens 17 Punkte erreicht
  • Kein Splitten, kein Verdoppeln, keine Versicherung
  • Unentschieden = keine Kapitaländerung

2.2 Modellannahmen

Zur Vereinfachung der Simulation haben wir folgende Annahmen getroffen:

  • Unabhängiges Kartenziehen: Jede Karte wird unabhängig aus dem vollständigen Deck gezogen. Wir führen kein fortlaufendes Deck — dies ist eine bewusste Vereinfachung, die den Vergleich zwischen Strategien nicht wesentlich beeinflusst.
  • Kein Blackjack-Bonus: Ein natürliches Blackjack (21 mit 2 Karten) wird 1:1 ausbezahlt, nicht 3:2.
  • Fixer Einsatz: Jeder Spieler setzt CHF 5 pro Runde (ausser Martingale).

2.3 Strategien

Wir vergleichen vier Strategien:

Strategie Beschreibung
Basic Hit wenn Summe < 17, Stand ab 17
Aggressive Hit wenn Summe < 19, Stand ab 19
Random Hit oder Stand zufällig (50/50)
Martingale Basic Hit/Stand; Einsatz nach Verlust verdoppeln, nach Gewinn zurücksetzen

2.4 Verwendete Wahrscheinlichkeitsverteilungen

Unser Modell enthält mehrere zufallsbasierte Grössen, die wir mit passenden Verteilungen beschreiben.

Diskrete Verteilung für den Rundenausgang Der Ausgang einer einzelnen Runde ist eine diskrete Zufallsvariable mit drei möglichen Ergebnissen: Gewinn (+5 CHF), Verlust (−5 CHF) oder Unentschieden (0 CHF). Diese drei Ausgänge entstehen durch die simulierten Spielregeln und werden empirisch aus den Monte-Carlo-Simulationen beobachtet.

Bernoulli-Verteilung Die Ruinwahrscheinlichkeit pro Spieler ist binär: Ein Spieler endet entweder pleite oder nicht. Das lässt sich mit einer Bernoulli-Verteilung modellieren, wobei der Parameter \(p\) die geschätzte Wahrscheinlichkeit für Ruin ist.

Geometrische Verteilung Die Anzahl überlebter Runden ist eine diskrete Zählvariable. Sie kann als empirische Verteilung betrachtet werden; je nach Strategie kann sie grob an eine geometrische Verteilung erinnern, muss aber nicht exakt geometrisch sein, weil unsere Simulation zusätzliche Regeln wie Einsatzgrenzen und Spielabbrüche enthält.

Normalverteilung Das Endkapital ist das Resultat vieler einzelner Runden mit Gewinn, Verlust oder Unentschieden. Für Strategien mit fixem Einsatz kann diese Verteilung bei vielen Wiederholungen annähernd glockenförmig wirken. Wir behandeln sie jedoch bewusst als empirische Verteilung aus der Simulation und nicht als exakt normalverteilt.

3.2 Handsumme berechnen

Code anzeigen 
# Ass = 11, wird zu 1 umgerechnet wenn Bust (Summe > 21)
calculate_sum <- function(hand) {
  total <- sum(hand)
  aces  <- sum(hand == 11)
  while (total > 21 && aces > 0) {
    total <- total - 10
    aces  <- aces - 1
  }
  total
}

3.3 Eine Runde spielen

Code anzeigen 
# Simuliert eine Blackjack-Runde
# Gibt "win", "loss" oder "draw" zurück
play_round <- function(strategy = "basic") {
  shuffled    <- sample(deck)
  player_hand <- shuffled[1:2]
  dealer_hand <- shuffled[3:4]
  player_sum  <- calculate_sum(player_hand)
  dealer_sum  <- calculate_sum(dealer_hand)

  # Spielerentscheidung je nach Strategie
  if (strategy == "basic" || strategy == "martingale") {
    while (player_sum < 17) {
      player_hand <- c(player_hand, sample(deck, 1))
      player_sum  <- calculate_sum(player_hand)
    }
  } else if (strategy == "aggressive") {
    while (player_sum < 19) {
      player_hand <- c(player_hand, sample(deck, 1))
      player_sum  <- calculate_sum(player_hand)
    }
  } else if (strategy == "random") {
    while (player_sum < 21) {
      if (sample(c(TRUE, FALSE), 1)) {
        player_hand <- c(player_hand, sample(deck, 1))
        player_sum  <- calculate_sum(player_hand)
      } else break
    }
  }

  # Dealer zieht immer bis 17 (Casino-Regel)
  while (dealer_sum < 17) {
    dealer_hand <- c(dealer_hand, sample(deck, 1))
    dealer_sum  <- calculate_sum(dealer_hand)
  }

  # Ergebnis bestimmen
  if (player_sum > 21)           return("loss")  # Spieler Bust
  if (dealer_sum > 21)           return("win")   # Dealer Bust
  if (player_sum > dealer_sum)   return("win")   # Spieler näher an 21
  if (player_sum < dealer_sum)   return("loss")  # Dealer näher an 21
  "draw"                                          # Gleichstand
}

3.4 Einen Spieler simulieren

Code anzeigen 
# Spieler spielt bis pleite oder maximale Rundenzahl erreicht
# Unentschieden = keine Kapitaländerung, kein Einsatzwechsel
simulate_player <- function(strategy = "basic",
                             start_capital = 100,
                             bet = 5,
                             max_rounds = 200) {
  capital         <- start_capital
  rounds          <- 0
  current_bet     <- bet
  capital_history <- c(start_capital)

  while (capital >= current_bet && rounds < max_rounds) {
    if (strategy == "martingale") {
      result <- play_round("martingale")
      if (result == "win") {
        capital     <- capital + current_bet
        current_bet <- bet                           # Einsatz zurücksetzen
      } else if (result == "loss") {
        capital     <- capital - current_bet
        current_bet <- min(current_bet * 2, capital) # Einsatz verdoppeln
      }
    } else {
      result <- play_round(strategy)
      if (result == "win")  capital <- capital + bet
      if (result == "loss") capital <- capital - bet
    }

    rounds <- rounds + 1
    capital_history <- c(capital_history, capital)
  }

  list(
    final_capital   = capital,
    rounds_played   = rounds,
    capital_history = capital_history
  )
}

3.5 Monte Carlo Simulation

Wir simulieren 10’000 Spieler pro Strategie — insgesamt 40’000 simulierte Spieler.

Code anzeigen 
set.seed(42)
n          <- 10000
strategies <- c("basic", "aggressive", "random", "martingale")
results    <- data.frame()
example_histories <- list()

for (strat in strategies) {
  final_capitals <- numeric(n)
  rounds_played  <- numeric(n)

  for (i in 1:n) {
    sim               <- simulate_player(strat)
    final_capitals[i] <- sim$final_capital
    rounds_played[i]  <- sim$rounds_played
    if (i == 1) example_histories[[strat]] <- sim$capital_history
  }

  results <- rbind(results, data.frame(
    strategy      = strat,
    capital       = final_capitals,
    rounds_played = rounds_played
  ))
}

4. Ergebnisse

4.1 Kennzahlen

Die Resultate zeigen deutliche Unterschiede zwischen den Strategien. In unserer Simulation schneidet die Basic-Strategie insgesamt am besten ab: Sie erreicht das höchste durchschnittliche Endkapital und die längste durchschnittliche Überlebensdauer bei vergleichsweise moderater Ruinrate. Die Aggressive-Strategie ist riskanter und bringt keinen klaren Vorteil. Die Random-Strategie zeigt erwartungsgemäss das schlechteste Ergebnis.

Die Martingale-Strategie zeigt ein typisches Muster von kurzen Gewinnphasen und seltenen, aber starken Verlusten. Dadurch wirkt sie kurzfristig günstig, hat aber in unserem Modell eine hohe Ruinwahrscheinlichkeit.

5. Fazit

Unsere Fragestellung war: Welche Blackjack-Strategie überlebt am längsten — und wie gross ist die Wahrscheinlichkeit, das Casino zu schlagen? Die Simulation zeigt, dass Basic unter den betrachteten Strategien am überzeugendsten abschneidet.

Trotzdem gelingt es keiner Strategie, das Casino langfristig zuverlässig zu schlagen. Martingale kann kurzfristig erfolgreich wirken, erhöht aber das Risiko eines schnellen Ruins deutlich. Random ist als Vergleichsstrategie nützlich, weil sie zeigt, wie schlecht eine unsystematische Entscheidung im Vergleich zu einer festen Regel abschneidet.

Eine wichtige Einschränkung ist, dass unser Modell vereinfacht ist: Wir verwenden eine vereinfachte Blackjack-Regel ohne Splitten, Verdoppeln oder Versicherungen, und das Kartenziehen ist ebenfalls vereinfacht modelliert. Dadurch sind die absoluten Resultate nicht 1:1 auf ein reales Casino übertragbar, die relative Gegenüberstellung der Strategien bleibt aber als Lernmodell gut interpretierbar.
